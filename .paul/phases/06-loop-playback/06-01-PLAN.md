<objective>
## Goal
Add sample looping with crossfade to Samplexpress. Users can set loop start/end points, enable/disable looping, and adjust crossfade length via the SAMPLE tab.

## Purpose
Loop playback is the core feature evolution for v0.1 completion, inspired by giada's real-time loop rendering. It enables sustained pad sounds, rhythmic loops, and continuous playback beyond the sample's one-shot duration.

## Output
- 4 new APVTS parameters (loop_enable, loop_start, loop_end, crossfade_ms)
- Real-time loop rendering with cosine crossfade in SamplexpressVoice
- Visual loop markers on WaveformDisplay
- Loop controls in SAMPLE tab (toggle + 3 sliders)
</objective>

    In renderNextBlock():
    - If loop enabled and sourceSamplePosition >= loopEndSample:
      - Calculate crossfade samples from crossfade_ms * sampleRate / 1000.0
      - During crossfade: mix current position with loopStartPosition + offset using cosine crossfade
      - After crossfade: jump to loopStartPosition + (position - loopEndPosition)
    - If loop enabled and sourceSamplePosition < loopStartSample (e.g. after note-on before loop region): play normally until reaching loop start
    - Loop start/end are converted from normalised 0–1 to sample indices in startNote() and cached as int samples
    - Use juce::dsp::FastMathApproximations::cos for crossfade curve

    Ensure loop logic doesn't break non-loop playback (loop_enable = false → existing path unchanged).
    Thread safety: parameter reads via std::atomic<float>::load() are lock-free and real-time safe.
  </action>
  <verify>Build succeeds; loop logic compiles without warnings</verify>
  <done>AC-2 satisfied: loop playback with crossfade works in real-time render</done>

- No one-shot mode ( Classic Mode only )
- No bidirectional looping (forward loop only)
- No loop fine-tuning via waveform drag (sliders only for v0.1)
- No crossfade curve options (cosine only)
- No loop beat-sync or tempo matching
</boundaries>
