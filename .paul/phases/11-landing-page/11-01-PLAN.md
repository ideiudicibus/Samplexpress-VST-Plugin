<objective>
## Goal
Ship a single-page static landing site for Samplexpress at `docs/index.html` plus a companion page at `docs/process.html` that summarises `docs/PROCESS-RETROSPECTIVE.md` with a generated SVG process diagram. Both pages are served via GitHub Pages and reflect the existing dark visual language of the plugin UI.

## Purpose
The README already references `docs/screenshot_v0.3_main.png` and `docs/architecture.png` — visitors arriving from search engines or GitHub need a real entry page rather than a raw README. The retrospective is the most interesting prose in the project and deserves its own page so that curious developers can find it without reading the GitHub folder structure.

## Output
- `docs/index.html` — single-page landing site (hero, features, screenshot, build, download, footer) with a link to `process.html`
- `docs/style.css` — stylesheet, plugin-faithful palette
- `docs/process.html` — retrospective summary page (process diagram + summary paragraph + link to the full MD)
- `docs/process-diagram.svg` — generated SVG visualising the PAUL PLAN → APPLY → UNIFY loop described in the retrospective
- `.github/workflows/pages.yml` — GitHub Pages deploy workflow
- `README.md` — updated to point at the deployed site
</objective>
